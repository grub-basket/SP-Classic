# 0.84.17 — Encrypted exports, loose-file import, persisted render cache

## Encrypted `.stash` export / import (0.84.3, 0.84.13–0.84.17)
`.stash` bundles can now be password-protected with **AES-256-GCM** and a
strong key derivation. Backward-compatible with unencrypted bundles.

- **Encryption suite:** Argon2id KDF (falls back to PBKDF2 in environments
  without Argon2), AES-256-GCM, authenticated header.
- The KDF that produced a given file (Argon2id vs PBKDF2) is surfaced in
  the export toast and import modal, so you know what's protecting it.
- **Encrypted exports are tagged in the filename** with `-encrypted` and a
  live preview during the export rename modal.
- **Import is notification-first.** Dropping an encrypted bundle no longer
  blocks the UI with a password modal — you get a toast that opens the
  password prompt on click. The modal gains a **Remind me later** action
  that reschedules the prompt instead of dismissing the bundle.
- **Manual-import bookkeeping:** a manual import clears the
  encrypted-waiting queue so "Remind me later" never resurfaces a bundle
  you've already brought in.

## Export rename modal (0.84.2)
- A native-undo-aware rename step before the file is written. The default
  name is prefilled and pre-selected so you can type a fresh name in one
  go; standard `Cmd+Z` works inside the modal.

## Loose-file + loose-folder import (0.84.1, 0.84.4, 0.84.5, 0.84.7–0.84.9)
Iteration on the import flow that landed in 0.79.

- **Command:** **Import loose files in the current folder** — manual
  counterpart to the auto-import watcher.
- **Loose folders too.** Pasted subfolders are converted into nested note
  trees, mirroring their on-disk structure (same shape as folder-drop in
  0.79.6).
- **Smarter folder import.** Reserved subfolders (`_archive`,
  `_attachments`, `_authors`) merge into the existing destination instead
  of being copied as nested notes. Existing note ids are preserved.
- Fixed a regression where folder-import adoption seeded an archived id
  as a collision (the kept-id case).
- **Shared `importLooseInto` primitive** so rebootstrap and the new
  command run identical logic.
- **Loose-import sweep handles `.stash` bundles** (decrypt-aware), and
  always skips `.edtz` at the top level *and* nested inside folder
  imports. `NON_NOTE_EXTENSIONS` is centralized.

## Auto-import a dropped `.stash` (0.84.10–0.84.12)
- Dropping a `.stash` bundle into a Stashpad folder root now auto-imports
  it, armed against the startup `create` storm.
- **Retroactive sweep:** at startup + every 5 minutes, every Stashpad
  folder is scanned for loose drops that arrived while Obsidian was
  closed.
- **Option C for encrypted bundles during a sweep:** rather than
  interrupt the user with a modal, encrypted bundles defer to a
  non-blocking snoozable toast so the rest of the sweep keeps going.

## Performance (0.83)
- **Skip recovery-link writes (setting).** Profiling showed `fmSync` (the
  `parentLink` / `children` recovery fields) running ~628ms per move on
  large vaults — pure overhead for users who don't rely on the recovery
  fields. New **Write recovery navigation links** toggle (default on)
  skips those writes entirely when off.
- **Persisted render cache.** A new `RenderCacheStore` backs the
  in-memory body-render cache with a single `render-cache.json` in the
  plugin private dir. A cold open does one bulk read to repopulate
  instead of re-rendering every visible body.

## Internal (0.82.5–0.82.9)
- `view.ts` split into five focused modules — `view-keys.ts`,
  `view-helpers.ts`, `authorship-tracker.ts`, `view-dnd.ts`,
  `note-body-renderer.ts`, `view-sort.ts`. No user-facing change, but
  navigation around the codebase is much easier and the modules cache
  better.
