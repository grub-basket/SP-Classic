# 0.99.14 — Encrypted notes no longer disappear on restart

Fixes a bug where a locked (encrypted) note's 🔒 placeholder could vanish after
restarting the app. The placeholders were rebuilt by scanning the vault's file
index for the encrypted blobs, but on a fresh start the view renders before the
vault finishes indexing those files — so the placeholders came up empty and the
registry that anchors them could be cleared.

Now the placeholders render from the saved registry (available immediately at
startup) and the registry is reconciled against the disk directly (and only
after the vault is fully loaded), so locked notes stay put across restarts. A
locked note whose registry entry had been lost is recovered from its sidecar on
the next launch.
