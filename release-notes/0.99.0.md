# 0.99.0 — Note clipboard + encryption hardening

## Copy, cut, and paste note blocks

A note clipboard that runs in parallel with the system clipboard, on the
familiar keys (Mod+C / Mod+X / Mod+V, rebindable in Settings → Hotkeys):

- **Copy** puts the note bodies on the system clipboard as text AND the notes
  on the note clipboard — paste in the list to **duplicate** them (fresh ids;
  paste again for more copies), or paste anywhere else for plain text.
- **Cut** + paste in the list **moves** the notes after the cursor row.
- **Cut** + paste in the **composer** drops the text in and deletes the
  original notes (undoable). This only triggers when the pasted text is
  actually the cut text — copying something else in between leaves the cut
  pending, so nothing is deleted by surprise.
- Copy/cut defer to the normal clipboard when text is highlighted; paste only
  intercepts when the note clipboard holds notes. Everything is undoable.

## Security hardening (post-release review of the encryption feature)

A dedicated security + code review of 0.98.38's vault encryption; all findings
fixed:

- The encrypted blob is now **verified on disk** (read back and byte-compared)
  before any plaintext is deleted — a truncated write on a network drive or
  sync client can no longer cost data.
- The idle auto-lock can no longer corrupt an in-flight lock operation (the
  session key is handed out as a copy, never the live buffer).
- Notes **edited while a lock is running are kept**, not deleted (the bundle
  holds the pre-edit content; a warning lists anything left in plaintext).
- A failed plaintext deletion now raises a persistent warning instead of
  silently reporting a clean lock.
- The persisted render cache no longer retains the text of locked or
  securely-deleted notes.
- Right-click → Delete now honors "Encrypt items sent to trash" (it previously
  bypassed the encrypted trash).
- Restore destinations from sidecar files are sanitized (a tampered sidecar
  can't redirect decrypted notes outside the vault), missing origin folders
  are recreated, and hidden-title deletes no longer reveal their origin folder
  on disk.
- KDF parameters in encrypted files are clamped, so a crafted file can't
  demand absurd memory/CPU when a password is entered.
- "Remove encryption" now scans the disk directly so it can never miss a
  freshly-written encrypted note.
- The "Encrypt trash filenames" and "Follow Obsidian's trash setting" toggles
  are now functional.

## Encrypted trash backfill

- New command: **"Encrypt existing Obsidian trash"** — sweeps everything
  already sitting in plaintext in the vault's `.trash/` into the encrypted
  trash. Restore puts the files back into `.trash/` exactly as they were.
  ("Encrypt items sent to trash" only covers new deletes; this covers the
  backlog.)

## Fixes & polish

- Archive and Encrypt-&-delete commands now have keybind rows in Settings →
  Hotkeys (and are findable via settings search).
- The encrypted-trash tab no longer duplicates rows when refreshed rapidly.
- Unlock/restore read note metadata from disk, fixing rare cases where a
  restored child note was re-attached to the top level.
- Pending archive sweeps are cancelled when the plugin unloads.
