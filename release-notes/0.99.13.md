# 0.99.13 — Tab management + note-clipboard polish

## Tabs

- **New command: “Close duplicate & orphaned Stashpad tabs (tidy up).”** Collapses
  duplicate tabs (keeping one per folder + focused note) and prunes tabs whose
  note no longer exists. It warms up every tab first so it works on the real
  views, keeps the active/healthy one, and shows a short summary of what it did.
- **Pinned-note clicks no longer hijack or duplicate tabs.** Clicking a pinned
  (or shared / task) note now reuses an existing tab on that folder if there is
  one — deferred tabs included — and only opens a new tab when there isn't.
- **Tab titles carry the note id** when focused into a note (`Folder — Title · id`)
  so tabs on same-titled notes stay distinguishable.

## Copy / cut / paste of notes

Building on the note clipboard, the two paste destinations are now clearly split
and work at any nesting depth:

- **Paste in the list = structural.** Copy → clones the note (and its subtree) at
  the current level; cut → moves it into the focused parent. Works whether or not
  a note is selected, and into empty parents.
- **Paste in a note’s composer = text.** Copy → drops the note’s text in; cut →
  drops the whole subtree’s text in (as an indented bullet outline) and deletes
  the originals. You can’t paste a cut note into itself.
- **Visual indicators:** cut notes ghost out; copied notes get a subtle tint.
  **Escape cancels** a pending cut/copy.
- **“Copy tree” (the outline command)** now also loads the note clipboard, so the
  whole stack can be pasted — cloned in the list, or dropped as an outline into a
  note.

## Import

- A `.stash` bundle can now carry **color-name aliases** (hex → friendly name);
  on import they’re added to the destination folder’s color settings, so colors
  set by an external tool come in already named.

## Fixes

- The in-list “Nest under…” picker now lands the cursor and selection on the
  destination note, instead of drifting to the next note after the list shifts.
- Pasting works correctly when focused inside a parent (previously it only
  reliably worked at the top level).
