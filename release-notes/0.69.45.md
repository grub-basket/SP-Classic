# 0.69.45 — Search modal fixes

## Search modal
- **Enter on highlighted result now picks reliably.** The previous handler
  chain wasn't actually firing SuggestModal's Enter handler; the chip handler
  now explicitly calls `selectActiveSuggestion` in its non-chip branch.
  (0.69.44)
- **Folder-picker Escape restores focus to the search input.** After closing
  a sub-picker spawned from the search modal, the re-opened modal now
  receives keyboard focus instead of leaving it on whatever Obsidian's
  modal-stack happened to restore to. (0.69.45)
- **`on: <time>` filters narrow to ±60s** around the specified moment instead
  of degenerating to the full day. `on: 9:45pm` used to return every note
  from today because the time component was dropped; now a 2-minute window
  applies when a time is present, falling back to the day window when only a
  date is given. (0.69.45)
