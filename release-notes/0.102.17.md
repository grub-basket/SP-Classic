# Stashpad Classic 0.102.17

Compatibility release: **Stashpad Classic now uses only APIs available in stable
Obsidian**, so it installs and runs on the latest stable release (1.13) and every
version back to 1.7 — no insider build required.

## What changed

- **Encryption fully removed.** All encryption features are gone — commands, the
  Encryption settings tab, folder/note context-menu options, archive-folder
  auto-encryption, the encrypted trash, and the OS-keychain (`secretStorage`)
  integration. Encryption was the feature most tied to newer/insider-only APIs.
- **No more newer-than-stable API usage.** Removed or reworked every call the
  community-plugin review flagged as newer than the plugin's supported Obsidian
  version (`secretStorage`, the async `workspace.revealLeaf` signature, and the
  1.13 declarative-settings `update()` refresh). Behavior is unchanged on the
  versions where those APIs existed; the plugin no longer *requires* them.
- **Safer HTML + cleaner source.** Cached note bodies re-hydrate through Obsidian's
  `sanitizeHTMLToDom` (no `createContextualFragment`), and a literal byte-order-mark
  in a regex is now escaped.

## Compatibility

`minAppVersion` stays at **1.7.0**, so Stashpad Classic installs on the widest range
of stable Obsidian, including the current 1.13 stable line.
