# 0.71.14 — JD Index Builder: area + folder support

Improvements to the JD Index Builder introduced in 0.71.

- **Range-area prefixes** like `10-19 Life Admin` now match the strict
  regex and slot into the hierarchy as proper areas.
- **Folder-only entries** are picked up. The scan walks folders as well as
  files, so JD areas / categories that exist only as folder containers
  (`10-19 Life Admin/`, `11 Me & My Family/`) show up in the index even
  when they have no matching note file. Folder rows render as plain text
  tagged `_(folder)_`; file rows stay as wikilinks.
- **Hierarchy resolution upgraded.** Categories nest under their containing
  area by numeric range `[lo, hi]`; IDs continue to use the dotted-parent
  rule. Synthetic ranges aren't auto-created — an empty `10-19` won't
  materialize a row.
- **Build creates Stashpad notes for folders too**, link-less and treated
  like synthetic parents, so the tree stays well-formed.
